Initial Overview

1\) Client Background -- A brief overview of the company, table of each
department with the headings (Department, Function). - P01\
\
1.1) Current Problems -- The current problems that the company is
intending to solve with the solution. Table with headings (Problem,
Impact, Intended Solution) - P01

1.2) Stakeholder Analysis -- A table detailing stakeholders (people who
will interact with the solution) -- Table with headings (Stakeholder,
Needs, Concerns) - P01

1.3) Proposed Solution -- An short overview of the solution -- brief
bullet points of each requirement of the proposed solution. - P01

1.4) Key Requirements -- Identify one key requirement (e.g access
levels) that the solution will need ot have and explain it with a short
justification of why + one table. - P01\
\
1.5) Technology Decisions -- A table of the initial decisions of which
technology stacks to use (e.g Architecture, Backend, Frontend, Database,
Deployment). Table Headings (Decision, Choice, Reason) - P01

1.6) Success Criteria -- Based on 1.4 (Key Requirements) - List of the
success criteria of the system -- Will it meet the basic functionality
requirements? -- List of each success criteria -- (SC1, SC2, SC3 etc)

Project Plan

2\) Project Scope -- Scope Statement (What the project is providing, who
the project is undertaken by, what each team member does) -- What is in
scope, what is out of scope. (What the prototype needs, what the
prototype does not need)

2.1) Project Objectives -- A table of each project objective (this
should basically just be the reverse of the company's "Current Problems
(1.1)" Table Headings (# (Number), Objective, Success Measure (how it
works), Trace to Brief "Current Problems" (1.1)

2.2) Prioritisation -- Must Have (A list of what the prototype needs
within the 30 development hours), Should Have (Documented but not built
until later prototypes), Could Have (Things that would be beneficial but
do not need to be documented fully), Won't have (things we specifically
do not include with justification)

2.3) Tools & Technologies -- Do the exact same as 1.5, just go into more
detail. Table Headings (Tool/Technology, Purpose, Justification)

2.4) Risk Register -- Table with headings: \# (Number), Risk,
Likelihood, Impact, Mitigation Strategy

2.5) Timeline -- Gantt Chart (see Project Gantt app), Phase overview --
Table Headings (Milestone, Target, Phase), then a detailed hours
breakdown for each of the chosen phases (to meet the 67 required hours)
